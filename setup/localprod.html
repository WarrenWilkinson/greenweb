Testing Production PasswordsÂ¶

You can locally test greenweb with your production passwords. Which is a good idea, if only to check that your passwords were applied.

The procedure is the same as in launching the development instance of greenweb, but run sh /vagrant/coldstart.sh prod instead of just sh /vagrant/coldstart.sh:

user@host:~/$ cd greenweb/dev
user@host:greenweb/dev/$ vagrant up
user@host:greenweb/dev/$ vagrant ssh
vagrant@vagrant$ sh /vagrant/coldstart.sh prod

The prod argument to coldstart.sh will ensure salt is setup to use the production pillar data. The script will also install the gpg keys you created in the previous chapter (hint: dev/prod_pubkey.gpg and dev/prod_privkey.gpg) to the correct location on the salt master.
